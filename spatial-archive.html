Process Log / 01

The NeRF
Training Pipeline

The network is given a sparse set of 2D captures and learns a continuous, coordinate-based radiance function — for any 3D point and viewing direction, it predicts a colour and density. Geometry is not stored explicitly; it is resolved through repeated optimisation against the input images. PSNR (peak signal-to-noise ratio) is logged at each interval as a proxy for reconstruction fidelity — it rises as the field localises coarse structure, then fluctuates while high-frequency detail (edges, occlusion boundaries, specular surfaces) is resolved against noise in the sample set.

note — PSNR is non-monotonic by design at this training scale: short-horizon spikes (≈ iter 100) reflect early overfitting to a single dominant viewpoint before the field redistributes density across the full camera set.

Spatial Ingestion
3DGS Baseline

Subject: Western Pacific No.713 locomotive, on-site capture. Overlapping frames are taken around the structure so that shared features can be matched across views. From these correspondences, structure-from-motion recovers each camera's extrinsic matrix (position + orientation) and intrinsic matrix (focal length, sensor offset), and triangulates a sparse point cloud — point_cloud.ply — used to seed the initial Gaussian primitives.

Pipeline_Note

  1. 01 — Capture overlapping frames around the subject, minimum ~60% overlap between neighbours.
  2. 02 — Detect and match features across frames to estimate relative camera pose.
  3. 03 — Triangulate matched features into a sparse point cloud; solve intrinsics/extrinsics jointly.
  4. 04 — Initialise Gaussian primitives at point-cloud positions; optimise colour, opacity, covariance.
